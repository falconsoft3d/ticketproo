Paginación

Parámetros:

  • page: Número de página (default: 1)
  • per_page: Registros por página (default: 50, máximo: 100)

Ejemplo con paginación:

curl -X GET \
  -H "X-API-Token: {{ table.api_token }}" \
  "{{ request.scheme }}://{{ request.get_host }}{{ table.get_api_url }}?page=2&per_page=10"

Respuesta (200 OK):

{
  "success": true,
  "table": "{{ table.name }}",
  "total": 45,
  "page": 1,
  "per_page": 50,
  "data": [
    {
      "id": 1,
      "data": { ... },
      "created_at": "...",
      "updated_at": "..."
    }
  ]
}

Errores Comunes

Causa: Token incorrecto o ausente

Solución:

  • Verifica que el token sea correcto
  • Confirma que estés usando el header X-API-Token
  • O usa el parámetro ?token=... en la URL

Causa: Datos no válidos

Solución:

  • Lee el array errors en la respuesta
  • Verifica que todos los campos requeridos estén presentes
  • Confirma que los tipos de datos sean correctos
  • Revisa las restricciones de longitud

Causa: El ID no existe

Solución:

  • Verifica que el ID del registro sea correcto
  • Confirma que el registro no haya sido eliminado
  • Lista los registros primero para ver los IDs disponibles
